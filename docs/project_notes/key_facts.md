When scaling beyond session pooler limits, options: (a) Prisma `previewFeatures = ["multiSchema"]` (prefixes `public.` in queries), (b) Supabase Team plan (configurable pooler search_path), (c) direct connection with Prisma-side pooling (`connection_limit` in URL). **DIRECT_URL** (for migrations) currently points to `pooler.supabase.com:5432` (session mode via pooler) вЂ” functional but ideally should point to `db.<project-ref>.supabase.co:5432` (true direct, no pooler).
